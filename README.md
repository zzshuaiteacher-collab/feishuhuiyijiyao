# 会议纪要工作提炼器

这是一个用于处理飞书会议纪要的 Codex Skill。

它不是普通的会议总结工具，而是把会议内容按“我的工作是否用得上”重新过滤，提炼成可行动、可沉淀、可写入文档的结构化结果。

## 适合解决的问题

当你每天有大量会议、交流、访谈或分享会纪要时，飞书智能纪要通常已经能给出总结，但它不会主动判断：

- 哪些内容和我的工作有关？
- 哪些观点可以转成产品功能、内容选题或客户洞察？
- 哪些信息值得沉淀进知识库？
- 哪些内容应该变成下一步行动？
- 哪些内容其实可以忽略？

这个 Skill 专门解决这个问题。

## 典型使用场景

- 读取一份飞书会议纪要，提炼与当前工作相关的内容。
- 从分享会中提炼可借鉴打法、案例和机会。
- 从客户访谈中提炼需求、痛点、异议和成交线索。
- 从项目会议中提炼决策、风险、分工和待办。
- 把会议内容沉淀成内容选题、产品想法、SOP 或客户洞察。

## 使用方式

把飞书会议纪要链接发给 Agent，并说明你希望提炼和什么工作相关。

示例：

```text
请用 meeting-work-extractor 读取这份飞书会议纪要，提炼和我的 AI Agent 工作流、内容生产、客户洞察相关的内容。
```

也可以直接说：

```text
这是一份会议纪要，帮我提炼跟我工作相关的内容，并输出可行动事项和可沉淀资产。
```

## 输入

必需：

- 飞书会议纪要、妙记、会议记录、访谈记录或文档链接。

可选：

- 当前工作画像。
- 当前项目名。
- 不关心的内容。
- 需要写入的目标飞书文档。

如果没有提供工作画像，Skill 会使用默认画像：

- AI Agent 工作流
- 飞书文档自动化
- 内容生产
- 商业诊断
- 客户洞察
- 获客
- 出海
- 小微企业增长

## 输出内容

默认输出包括：

- 一句话结论
- 与我工作相关的内容
- 可转成行动的事项
- 可沉淀为资产的内容
- 待验证问题
- 暂不处理内容

输出重点不是“会议讲了什么”，而是“这场会对我的工作有什么用”。

## 输出示例

```markdown
# 会议工作提炼：AI 外贸获客及创业经验分享会

## 一句话结论
这场会最值得沉淀的是：AI 不应只用于效率提升，还可以用于获客流程重构、市场洞察和业务资产积累。

## 与我工作相关的内容
| 内容 | 为什么相关 | 可用在哪里 | 可信度 |
|---|---|---|---|
| 用 AI 监控同行评论区找客户线索 | 可转成获客 Agent 或销售线索流程 | 产品功能 / SOP | 中 |

## 可转成行动的事项
- [ ] 设计一个“会议纪要到工作资产”的固定输出模板
  - 来源：会议中多次出现 AI 工作流和业务落地案例
  - 下一步：先用 3 份会议纪要测试模板稳定性
  - 负责人/时间：待定
```

## 设计原则

- 不重复飞书智能纪要已有总结。
- 每条提炼都说明为什么和工作相关。
- 优先输出能进入下一步工作的内容。
- 不把所有会议内容都塞进结果。
- 写入文档时优先追加，不覆盖原文。
