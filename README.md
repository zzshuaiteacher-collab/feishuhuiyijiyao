# 会议纪要工作提炼器

这是一个用于处理飞书会议纪要的 Codex Skill。

它不是普通的会议总结工具，而是把会议内容按“我的工作是否用得上”重新过滤，提炼成可行动、可沉淀、可写入文档的结构化结果。

## 适合解决的问题

当你每天有大量会议、交流、访谈或分享会纪要时，飞书智能纪要通常已经能给出总结，但它不会主动判断：

- 哪些内容和我的工作有关？
- 哪些观点可以转成产品功能、内容选题或客户洞察？
- 哪些信息值得沉淀进知识库？
- 哪些内容应该变成下一步行动？
- 哪些内容其实可以忽略？

这个 Skill 专门解决这个问题。

## 典型使用场景

- 读取一份飞书会议纪要，提炼与当前工作相关的内容。
- 从分享会中提炼可借鉴打法、案例和机会。
- 从客户访谈中提炼需求、痛点、异议和成交线索。
- 从项目会议中提炼决策、风险、分工和待办。
- 把会议内容沉淀成内容选题、产品想法、SOP 或客户洞察。

## 使用方式

把飞书会议纪要链接发给 Agent，并说明你希望提炼和什么工作相关。

示例：

```text
请用 meeting-work-extractor 读取这份飞书会议纪要，提炼和我的 AI Agent 工作流、内容生产、客户洞察相关的内容。
```

也可以直接说：

```text
这是一份会议纪要，帮我提炼跟我工作相关的内容，并输出可行动事项和可沉淀资产。
```

## 输入

必需：

- 飞书会议纪要、妙记、会议记录、访谈记录或文档链接。

可选：

- 当前工作画像。
- 当前项目名。
- 不关心的内容。
- 需要写入的目标飞书文档。

如果没有提供工作画像，Skill 会使用默认画像：

- AI Agent 工作流
- 飞书文档自动化
- 内容生产
- 商业诊断
- 客户洞察
- 获客
- 出海
- 小微企业增长

## 输出内容

默认输出包括：

- 会议关键内容
- 核心关键
- 跟工作的关系
- 一句话总结

输出顺序是：先讲清楚“会议说了什么”，再讲“跟我有什么关系”。不要一上来就给结论或表格，结论要直接说人话。

## 输出示例

```markdown
# 会议工作提炼：AI 外贸获客及创业经验分享会

## 📋 会议关键内容
这场会主要讨论 AI 怎么真正进入外贸获客和出海业务，而不是只停留在工具体验。会上有做 AI 外贸获客的人分享了如何用 AI 找线索、做触达和提升效率，也有人从创业角度提醒大家，业务本身不赚钱时，AI 不能替你解决方向问题。

## 🎯 核心关键
这场会的本质是在讨论：AI 应该服务于真实业务增长，而不是变成新的工具崇拜。

## 💼 跟工作的关系
1. 可以把“AI 外贸获客流程”拆成一个 Agent 工作流，用来做线索发现、客户判断和触达话术。
2. 可以把会上关于“先看业务是否成立，再谈 AI 降本”的观点沉淀成商业诊断内容。
3. 可以用这场会测试“会议纪要到工作资产”的固定提炼模板。

## ⚡ 一句话总结
这场会说的是：别为了用 AI 而用 AI，要让 AI 帮你找到客户、看懂业务、沉淀资产。
```

## 设计原则

- 不重复飞书智能纪要已有总结。
- 先讲清楚会议说了什么，再讲跟工作有什么关系。
- 结论直接说人话，不一上来就给表格。
- 优先输出能进入下一步工作的内容。
- 不把所有会议内容都塞进结果。
- 写入文档时优先追加，不覆盖原文。
