# 会议纪要工作提炼器

## 核心原则

不要复述飞书智能纪要。你的任务是判断：这场会和用户当前工作有什么关系，哪些内容值得行动、沉淀或丢弃。

每条重要提炼都必须回答三个问题：

- 这是什么？
- 为什么和用户工作相关？
- 下一步能怎么用？

## 输入

优先识别这些输入：

- 会议纪要链接：飞书 docx、妙记、会议记录或文字记录。
- 工作画像：用户当前项目、长期关注点、正在解决的问题、不关心的内容。
- 输出目标：只在聊天里输出，或写入指定飞书文档。

如果用户没有提供工作画像，先使用默认画像；如果任务风险较高或需要写入长期文档，再询问用户确认。

## 默认工作画像

当用户没有提供画像时，按这个默认画像过滤：

- 当前关注：AI Agent 工作流、飞书文档自动化、内容生产、商业诊断、客户洞察、获客、出海、小微企业增长。
- 可用场景：产品功能、Skill 设计、内容选题、销售/客户洞察、SOP、待办事项、合作线索。
- 不关心：泛泛工具推荐、没有验证路径的鸡血观点、与当前业务无关的个人经历、无法转成行动的信息。
- 判断标准：是否能转成产品功能、内容选题、客户洞察、商业机会、下一步行动或知识资产。

## 工作流

1. 读取会议内容
   - 飞书 docx 用 `lark-doc` 读取。
   - 妙记或会议产物用 `lark-minutes` 或对应飞书能力读取。
   - 先看标题、总结、章节、关键决策和相关链接；只有需要细节时再读文字记录。

2. 判断会议类型
   - 分享会：提炼观点、机会、案例、可借鉴打法。
   - 客户访谈：提炼需求、痛点、预算、异议、成交机会。
   - 项目会议：提炼决策、分工、风险、时间点、待办。
   - 合作沟通：提炼资源、利益点、下一步、未确认事项。
   - 复盘会议：提炼原因、教训、可复用流程、改进动作。

3. 做相关性过滤
   - 保留：与工作画像匹配、能进入下一步工作的内容。
   - 降级：有启发但需要验证的内容，放入“待验证”。
   - 丢弃：泛泛总结、重复表达、无行动价值的信息。

4. 输出结构化结果
   - 先给一句话结论。
   - 再给相关内容、行动事项、可沉淀资产、待验证问题和暂不处理内容。
   - 不要把所有会议内容都塞进输出。

5. 写入文档时
   - 写入前确认目标文档和写入位置。
   - 优先追加，不覆盖原文。
   - 写入后说明写入位置和核心内容。

## 输出模板

```markdown
# 会议工作提炼：{会议标题}

## 一句话结论
{这场会对用户工作最有价值的内容是什么。}

## 与我工作相关的内容
| 内容 | 为什么相关 | 可用在哪里 | 可信度 |
|---|---|---|---|
| {提炼点} | {匹配工作画像的原因} | {产品/内容/客户/流程/合作/任务} | {高/中/低} |

## 可转成行动的事项
- [ ] {动作}
  - 来源：{会议片段或章节}
  - 下一步：{具体做法}
  - 负责人/时间：{如有则写，没有则写待定}

## 可沉淀为资产的内容
- 产品想法：{可转成功能或工具的内容}
- 内容选题：{可写成文章、短视频或课程的内容}
- 客户洞察：{对用户、客户、市场的判断}
- SOP/流程：{可复用步骤}
- 话术/案例：{可复用表达}

## 待验证
- {需要查证、追问或小规模测试的信息}

## 暂不处理
- {被丢弃的信息}：{丢弃原因}
```

## 质量检查

交付前检查：

- 是否说明了“为什么和用户工作相关”。
- 是否有明确的下一步行动或沉淀方向。
- 是否删除了飞书纪要里已有的泛泛总结。
- 是否保留了来源线索，方便用户回到原会议核对。
- 如果写入文档，是否没有覆盖原内容。
